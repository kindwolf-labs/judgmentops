# Microsoft Submission User Guide

## Status

JudgmentOps is submission-ready for Microsoft for Startups, pending user eligibility confirmation and manual submission.

This guide does not establish eligibility or guarantee approval. The user must choose the truthful applicant identity and confirm legal, employment, intellectual-property, account, and billing considerations before submitting.

## Official Entry

- Microsoft for Startups landing page: https://www.microsoft.com/en-us/startups
- Microsoft for Startups application entry: https://startups.microsoft.com
- Official getting-started documentation: https://learn.microsoft.com/en-us/startups/microsoft-for-startups/getting-started-mfs

Application process:

1. Visit the Microsoft for Startups website.
2. Click **Get started**.
3. Sign in with the appropriate Microsoft account.
4. Choose the applicable path:
   - If affiliated with a participating investor, accelerator, incubator, university, or similar organization, use its valid Investor Network referral code.
   - If there is no referral code, use the startup path currently available without an Investor Network code. Do not claim an investor affiliation.
5. Fill in the startup and project information using the copy-ready materials below.
6. Review every field for accuracy before manually submitting.

Microsoft states that the program provides startup credits, Azure AI capabilities, technical resources, startup guidance, and go-to-market support. Current Microsoft documentation says credits can increase over time based on verified progress, service adoption, sustained Azure usage, and eligibility, with up to $150,000 available at higher stages. A referral-code offer and the open startup-credit path have different eligibility and benefit levels.

Do not use generic Microsoft support, Xbox support, Azure billing support, or consumer support as an application route.

## Eligibility Confirmation Checklist

The user must confirm:

- [ ] What legal entity or project identity will be used.
- [ ] Whether there is a company website or public project site.
- [ ] Whether the public GitHub repository will be used as the project website if no separate site exists.
- [ ] The truthful country or region of the applicant and project.
- [ ] Whether there is a valid Investor Network referral code from an existing affiliated organization.
- [ ] Whether JudgmentOps can honestly be described as a startup or early-stage open-source infrastructure project under the selected application path.
- [ ] Whether the Microsoft account used for submission is appropriate and controlled by the applicant.
- [ ] Whether creating or linking an Azure account and accepting its billing terms is acceptable.
- [ ] Whether any employer agreement, intellectual-property obligation, conflict-of-interest rule, or outside-work policy affects the project or application.
- [ ] Whether all company, team, funding, incorporation, and traction answers can be provided truthfully.

This checklist is operational guidance, not legal advice. The user should decide whether professional legal or employment-policy review is needed.

## Recommended Positioning In The Form

Use startup and product-infrastructure framing, not research-only framing.

Preferred:

> JudgmentOps is an early open-source infrastructure project for making long-running AI-assisted software work auditable, repeatable, and reviewable.

Avoid:

- "Personal AI OS."
- "Prompt library."
- "Smart contract."
- "Finished benchmark."
- "Approved by OpenAI, Meta, or Microsoft."
- Internal, private, or mystical language.

## Submission Notes

- If asked for traction, state honestly: early public prototype, v0.1.2 release, schemas, 20 cases, worked examples, local CLI, paired fixtures, a thin adapter, and reviewer materials.
- Do not invent users, revenue, customers, downloads, stars, partnerships, or community size.
- If asked for funding stage, answer truthfully.
- If asked for team or company status, answer truthfully and consistently with the selected legal or project identity.
- If asked for development stage, choose the closest truthful option. The public repository supports an early prototype description; do not imply a mature product.
- If asked how Azure would be used, emphasize the evaluation harness, GitHub-native workflow integration, Azure-hosted case explorer, CI and documentation hosting, and Azure AI model runs.
- If asked about go-to-market, mention open-source maintainers, developer teams using coding agents, AI platform teams, and enterprise workflow-governance needs.
- If asked about the requested credit amount, do not assume eligibility for $150,000. Microsoft describes higher credits as progressively unlocked or tied to eligible investor offers.
- Review spelling, URLs, project identity, and every factual claim before submitting. Microsoft documentation states that a submitted application may not be directly editable.

## After Submission

Record:

- Submission date.
- Microsoft account used for submission, without storing sign-in details.
- Selected application path: referral-code investor offer or no-code startup-credit path.
- Application ID or confirmation reference, if provided.
- The exact field versions submitted.
- Screenshot or PDF of the confirmation page, if available and safe to retain.
- Expected review or follow-up date.
- Any requested verification or next steps.

Then update `docs/SUBMISSION_TRACKER.md`:

- Change Microsoft submission status to `Submitted`.
- Add the actual submission date.
- Add the follow-up date or review expectation.
- Note the selected path without recording sensitive account data, referral codes, or authentication material.

## User Decision Required

Choose one:

- Submit now, after completing the eligibility checklist.
- Defer until the Google package is ready.
- Submit after checking employer, intellectual-property, legal-entity, account, and billing comfort.

Recommended default: clarify eligibility first if the legal or project identity, employer/IP position, referral-code status, or Azure billing setup is uncertain. Otherwise, the application materials are ready for manual submission.
